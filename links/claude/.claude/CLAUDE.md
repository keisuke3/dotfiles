# Claude 開発ガイドライン

## 基本原則
- 言語: ユーザーへの応答は日本語で行ってください。
- 簡潔性: 前置きや社交辞令を省き、技術的な解決策を直接的かつ簡潔に提示してください。
- 思考プロセス: 複雑なタスク（3ファイル以上の変更やアーキテクチャの変更）の場合は、コードを書く前に短い「計画（Plan）」を提示し、合意を得てから実装に移ってください。

## システム環境とツールチェーン
- パス操作: 現在の作業ディレクトリをプロジェクトルートとして扱います。
- システムディレクトリ（/usr, /etc）や、プロジェクト外のディレクトリへの変更は、明確な指示がない限り行わないでください。

## 開発ツール標準
- パッケージ管理: Node.js: pnpm を優先利用（なければ npm）。yarn はユーザー指定がない限り使用しない。
- Python: グローバルな pip は禁止。常に venv または poetry 環境下での実行を確認すること。
- エディタ: コードの提案は VS Code での閲覧・編集を前提としてください。

## MCPの使用
- プロジェクトにSerena MCPが設定されている場合、コードの探索と分析には常にSerena MCPを使用してください。
- ライブラリ/API ドキュメント、コード生成、セットアップ、または構成手順が必要な場合は、常に Context7 MCP を使用してください。

## セキュリティ
- 秘密情報: APIキー、パスワード、認証トークンは絶対にチャットに出力しない、およびコードにハードコードしないでください。常に .env ファイルの使用を提案してください。
- 破壊的コマンド: rm -rf、DROP TABLE、git push --force などの不可逆的な操作は、実行しないでください。
- ファイルアクセス: .env、SSH鍵、AWS設定ファイルの中身を読み取る必要がある場合は、勝手に読み込まず、ユーザーに許可を求めてください。

## コーディング・エンジニアリング規約
- コード品質: DRY & SOLID: 重複を避け、単一責任原則を守ってください。
- 型安全性: TypeScriptやPythonのType Hintなど、型定義が利用可能な場合は厳密な型付けを行ってください。
- エラー処理: 例外を握りつぶさず、文脈情報を含めてログ出力またはスローしてください。
